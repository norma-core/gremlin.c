> 🚧 **Active development.** Primitive APIs and contract shapes are
> stable, but the `_at` fast-path family and the compile-time float /
> double helpers are still settling. Pin a commit if you depend on it.

`gremlin` is the 🔧 wire-format runtime underpinning the whole stack —
a header-only C99 library of protobuf encode/decode primitives
(varints, fixed32/64, zigzag, LEN_PREFIX bytes, wire-type skip) with
an ACSL contract on every function, all discharged by 🔬
[Frama-C WP](https://www.frama-c.com/).

Everything is a `static inline __attribute__((always_inline))` header
function. There is no `.a` artefact, no `.o`, no runtime link. Code
generated by [`gremlinc`](../gremlinc/README.md) `#include`s this
header and the compiler folds the primitives into the call sites.

## 🎯 Scope and non-goals

**✅ What this library is:**
- A wire-format primitive library — varint (LEB128) encode/decode with
  sign-extension, fixed32/64 little-endian encode/decode, zigzag
  round-trip, LEN_PREFIX bytes, wire-type skip.
- Header-only — one `gremlin.h`, no library artefact, no link step.
  The compiler sees the full body at every call site.
- Zero-allocation — every primitive operates on a caller-provided
  `struct gremlin_writer { uint8_t *buf; size_t cap; size_t offset; }`
  for encode, or a raw `(buf, len)` slice for decode.
- Verified — every primitive has an ACSL contract, byte-level where
  meaningful (e.g. `varint_byte(v, k)` relates each written byte to
  the canonical LEB128 encoding of `v`).

**❌ What this library is not:**
- A protobuf message encoder / decoder. Message shapes come from
  [`gremlinc`](../gremlinc/README.md) codegen; this library only
  provides the bit-level primitives codegen calls.
- A proto parser. That's [`gremlinp`](../parser/README.md).
- A descriptor library. That's [`gremlind`](../descriptors/README.md).

## 🧰 What's in the header

Grouped by family — every entry is a `static inline` function in
`gremlin.h`.

| Family | Functions | Purpose |
| --- | --- | --- |
| 🪢 Writer | `writer_init` | Set up `{buf, cap, offset}` cursor. |
| 📏 Varint size | `varint_size`, `varint_size_cascade`, `varint_size_clz`, `varint32_size` | Compute LEB128 byte count for a value. `_clz` fast path uses `__builtin_clzll`; `_cascade` is the verification fallback. |
| ✍️ Varint encode | `varint_encode`, `varint32_encode`, `varint_encode_at`, `varint32_encode_at` | Write LEB128 at the current writer cursor (or at a caller-held `(buf, off)` pair for the `_at` register-hoisted fast path). |
| 📖 Varint decode | `varint_decode`, `varint32_decode` | Decode LEB128; `_32` bails after 5 bytes to diagnose overflow early. |
| 🔢 Fixed32/64 | `fixed32_size`, `fixed32_encode`, `fixed32_decode`, `fixed64_*`, `fixed32_encode_at`, `fixed64_encode_at` | Little-endian 4- and 8-byte values. `_at` variants mirror the varint register-hoisted fast path. |
| 📦 Bytes | `write_bytes`, `write_bytes_at`, `bytes_decode` | LEN_PREFIX write (length varint + raw bytes) and matching decode. |
| ⏭️ Skip | `skip_data` | Advance past an unknown wire-typed field's payload — used by readers to tolerate unexpected tags. |
| 🪞 Zigzag | `zigzag32`, `unzigzag32`, `zigzag64`, `unzigzag64` | Signed-varint round trip (`sint32` / `sint64`). |
| 🎱 Float / double bits | `f32_bits`, `f32_from_bits`, `f64_bits`, `f64_from_bits` | Type-punning IEEE-754 ↔ integer bit patterns (the one place the runtime uses a `union`). |

## 📚 Using the library

The intended consumer is [`gremlinc`](../gremlinc/README.md) — the
protobuf message codegen emits calls to these primitives and includes
`gremlin.h` at the top of every generated `.pb.h`. If you want to
hand-write encoders, the header is self-documenting; every function
carries an ACSL contract above the body that states its pre- and
post-conditions exactly.

## 🔨 Building

Dependencies: a C99 compiler and CMake ≥ 3.10.

From this directory:

```bash
mkdir build && cd build
cmake ..
make
```

Or from the [top-level repo root](../README.md) — the runtime is picked
up via `add_subdirectory(runtime)`.

The library is header-only (`add_library(gremlin_runtime INTERFACE)`),
so there is no artefact to install beyond the header itself.

Artifacts:
- 🧪 `float_test` — runtime sanity check for the `f32_bits` / `f64_bits`
  union type-punning (which WP can only prove memory-safe, not
  IEEE-754-faithful).

## 🔬 Formal verification

### 📊 Current status

**All 719 proof obligations pass ✅, plus 116 smoke tests:**

| Group              | Proved      | Smoke     | Notes                             |
| ------------------ | ----------- | --------- | --------------------------------- |
| 🛠️ `verify-runtime` | 719 / 719   | 116 / 116 | Every primitive in `gremlin.h` |

**Smoke tests are mandatory.** Frama-C's `-wp-smoke-tests` is wired
into `WP_BASE_FLAGS` and detects unreachable code, contradictory
contracts, and poisoned axiomatic models.

⚡ Subsequent runs only re-prove changed goals thanks to
`-wp-cache update`.

### ✅ What is proven

For every primitive, WP proves:

- 🛡️ **Memory safety** — no OOB reads, no null derefs, no integer
  overflow in offset/length arithmetic (via `-wp-rte`).
- ➡️ **Offset monotonicity** — `w->offset` only advances; on a
  successful encode, by exactly `varint_size(v)` / `4` / `8` / `len`.
- 📐 **Byte-level correctness** — each byte written by `varint_encode`
  matches `varint_byte(v, k)`, the canonical LEB128 byte function
  defined by three axioms in the `GremlinVarintBytes` axiomatic
  block. `fixed32_encode` byte output is tied to `v >> 8k`.
- 🔄 **Decode round-trip** — when `varint_decode` reports `GREMLIN_OK`,
  the returned value reconstructs the original (bounded by 1..10
  consumed bytes for 64-bit varint, 1..5 for 32-bit).
- 🧮 **Size / encode / decode consistency** — `varint_size(v) == k`
  is the universal size statement; encode advances by that much,
  decode consumes that much.
- ⚡ **CLZ fast path equivalence** —
  `varint_size_clz(v) == varint_size_cascade(v)` for all `v >= 1`, via
  the admitted bridge axiom `clz_matches_varint_size`.

### ⚠️ What is admitted (and why)

The trust base is very small:

**1. `__builtin_clzll` semantics** — `logic_clzll_range`,
`clz_matches_varint_size`
We declare an ACSL logic function for `__builtin_clzll` and two
axioms: one bounding its return value to `[0, 63]` for non-zero
input, another tying `(64 - clz(v) + 6) / 7` to `varint_size(v)`.
These are admitted because the intrinsic is opaque to Frama-C. The
bridge axiom is trivially true (it's the canonical bit-length
formula) and is the same admitted identity every LEB128
implementation relies on.

**2. Float / double bit-cast** — `f32_bits`, `f32_from_bits`,
`f64_bits`, `f64_from_bits`
ACSL proves the unions are memory-safe. Whether the bit patterns
line up with IEEE-754 is checked at runtime by `tests/float_test.c`,
which round-trips representative edge-case floats and asserts on
mismatch.

**3. `write_bytes` byte-level postcondition**
The memcpy fast path would need a libc-stub contract WP doesn't
reliably supply; we ship a byte-loop fallback and WP proves memory
safety. Byte content equality is documented inline in the contract
as a trust-base claim.

These admitted facts are small, local, and orthogonal to the
protobuf semantics; they don't hide bugs in the primitives.

### ❓ What is not verified

- 🧪 **The tests** in `tests/` — `float_test` and `verify.c` are
  regular C code, the former for runtime IEEE-754 checks (which
  complement WP) and the latter for `$FRAMAC -wp-fct` target
  enumeration.
- 🧵 **Thread safety** — writers are caller-owned; the header makes no
  multi-thread claims. Multiple readers over a `const *` source are
  safe in practice but not formally speced.

### 🛠️ Running verification

You need Frama-C with the WP plugin plus Alt-Ergo and Z3. See the
[parser README](../parser/README.md#️-running-verification) for full
install instructions — the setup is identical.

From the build directory:

```bash
make verify-runtime    # the full runtime proof
```

📂 Proof results are cached in `<repo>/runtime/.wp-cache/`; cold-cache
run takes a few minutes.

To re-prove from scratch:

```bash
rm -rf ../.wp-cache && make verify-runtime
```
